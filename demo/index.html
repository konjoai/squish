What is squish?

A KV cache that fits where models don't.

Modern LLMs spend more memory on the KV cache than on weights at long context. squish quantizes K and V on a per-token, per-channel basis with a Hadamard rotation in front, so the post-rotation distribution flattens and small codebooks (4 levels for INT2, 16 for INT4) survive the 0.1 % outlier channels that normally collapse low-bit quantization. The result: drop-in QuantizedKVCache / HadamardKVCache classes that plug into the mlx_lm cache protocol with a single mode argument.

Capabilities

One API. Three storage tiers.

INT8 per-token

The familiar baseline. ~44 dB SNR, 132 B/token at d=128. Zero quality cost — use this for short-context production traffic.

INT4 per-token NEW

16-level uniform codec, nibble-packed 2-per-byte. Half the memory of INT8, ~22 dB SNR — the right default for the 8 K–16 K context band where INT2 is overkill.

head_dim/2 + 4 B/tok≈ 22 dB
W104 · 2026-05
▤

INT2 per-token

4-level NF2 codebook bit-packed 4-per-byte. ~4× memory reduction, designed for ≥ 16 K contexts where the SNR cost is dominated by what Hadamard rotation already buys you.

head_dim/4 + 4 B/tok≈ 6 dB
QuaRot
◈

Hadamard rotation

HadamardKVCache rotates K/V with a random Walsh–Hadamard matrix before quantization, then un-rotates on read. Outlier energy spreads uniformly across dims; INT2 stops collapsing. Re-uses the existing seed-driven build path from squish.kv.kv_cache.

orthogonal · O(d log d)+1 to +12 dB
helper
◉

recommended_kv_mode_3tier

Auto-pick the storage mode by planned context length. Defaults: ≤ 8 K → INT8, 8 K–16 K → INT4, > 16 K → INT2. Thresholds and modes are overridable; degrades cleanly to the W104 2-tier helper.

int8 → int4 → int21-line setup

SnapKV importance scoring, NVMe disk-tier overflow, SVD KV compression, CommVQ, Q-Filters — all live alongside the quant tier. Construction-time guards keep the combinations that actually work from the ones that would silently break.

Snippets

Drop-in. One mode argument.

32 K context — Qwen2.5-7B python 
from squish.kv.kv_cache import (
    HadamardKVCache,
    recommended_kv_mode_3tier,
)

# pick storage by planned context length
mode = recommended_kv_mode_3tier(32_000)   # → 'int2'

cache = HadamardKVCache(
    n_layers=28,
    window=128,
    mode=mode,
    seed=42,
)

# same protocol mlx_lm uses — no other changes
model.generate(prompt, kv_cache=cache)
measure your own SNR python 
import numpy as np
from squish.kv.kv_cache import (
    _quantize_int4_per_channel,
    _dequantize_int4_per_channel,
)

v   = np.random.randn(512, 128).astype(np.float16)
q, s = _quantize_int4_per_channel(v)
vh  = _dequantize_int4_per_channel(q, s, 128)

err  = np.mean((v - vh)**2)
snr  = 10 * np.log10(np.mean(v**2) / err)
print(f"INT4 SNR = {snr:.2f} dB")
three-tier auto mode python 
from squish.kv.kv_cache import recommended_kv_mode_3tier

recommended_kv_mode_3tier(4_000)   # 'int8'  — short context
recommended_kv_mode_3tier(12_000)  # 'int4'  — medium
recommended_kv_mode_3tier(32_000)  # 'int2'  — long

# override thresholds
recommended_kv_mode_3tier(
    12_000,
    short_threshold=4_096,
    long_threshold=32_768,
)                                # 'int4'
raw INT2 codec python 
from squish.kv.kv_cache import (
    _quantize_int2_per_channel,
    _dequantize_int2_per_channel,
)

# 4-level NF2 codebook · indices packed 4-per-uint8
packed, scale = _quantize_int2_per_channel(activations)

# packed.shape == (n_tokens, head_dim // 4)
# packed.dtype == np.uint8                 
# scale.shape  == (n_tokens,) float32       

recon = _dequantize_int2_per_channel(packed, scale, 128)
